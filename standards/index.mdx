Glide ships eight JSON Schema documents at the public standards URL [`schemas.glide.dev/agent-banking/draft/`](https://schemas.glide.dev/agent-banking/draft/). Each schema is auto-generated from the corresponding Zod schema in `@glide/schemas` (or `@repo/connectors-base` / `@glide/skills-base`) via the `pnpm --filter @glide/schemas build-schemas` pipeline.

<Warning>
  All schemas currently publish under `/draft/`. Per the OSS plan §M5.5 Codex review fix #8, schemas pin to `/draft/` until **3+ independent implementers** have verifiably built against the draft. Promotion to `/v1/` is an explicit announced event — not implicit on time. This protects integrators from being locked into our first-draft mistakes.
</Warning>

## Versioning contract

- **`/draft/`** — current channel. Breaking changes ship without notice (we're iterating on shape).
- **`/v1/`** — locked once 3+ independent implementers verified. Breaking changes require `/v2/` with both versions live + a deprecation window.
- **`/v1.1/`** — additive-only updates within v1 (added optional fields).

The `/v1/` path is reserved permanently. Any schema-breaking change post-v1 promotion becomes a migration cost for every integrator — that's the deliberate cost of the lock.

## Generating + uploading

Schemas are auto-generated from the Zod source via:

```bash
pnpm --filter '@glide/schemas' build-schemas
# emits dist/schemas/draft/*.json
```

Glide Cloud uploads `dist/schemas/draft/` to Cloudflare Pages on every release. Self-hosters who want their own standards endpoint follow the same pipeline against their fork.

## Citing schemas

Each schema has a canonical `$id`:

```json
{
  "$id": "https://schemas.glide.dev/agent-banking/draft/connector-manifest.json",
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "channel": "draft",
  "schema": { ... }
}
```

Cite by `$id` in your own integration docs. The URL resolves to a real document.

## Standards governance

The `PUBLISHED_SCHEMA_NAMES` vocabulary (the closed set of names published as standards) is CODEOWNERS-protected. Adding a new standard schema is a public-API change requiring counsel + maintainer sign-off.
